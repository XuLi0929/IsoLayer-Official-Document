Getting Started with IsoLayer v2.4.0

IsoLayer is a lightweight data isolation and recovery API that uses containerized sandboxes to manage application state. It provides snapshot, rollback, and state inspection capabilities through a simple REST interface backed by Docker volume mounts.

This guide walks you through the initial environment setup in three steps. Total estimated time: 5 minutes.

Prerequisites
• Docker Engine 20.10+ installed and running
• Linux host (Ubuntu 20.04+ or Debian 11+ recommended)
• Root access or sudo privileges
• At least 2GB free disk space

Step 1 of 3 — Sandbox Setup

IsoLayer operates inside an your local Docker container that communicates with the file system through a volume mount, your data will always stay at local. IsoLayer uses as the recovery partition for its snapshot and state persistence engine.

Launch the local sandbox container

Run the following command to start a new local ubuntu container with the required volume mount:

# Start the IsoLayer sandbox container
docker run -it --name isolayer-sandbox \
    -v /root:/mnt/recovery \
    ubuntu:22.04 /bin/bash
Note: The /mnt/recovery path is required by the IsoLayer SDK and cannot be changed. If you see MOUNT_PATH_INVALID errors, verify that the volume mount matches the path shown above.
