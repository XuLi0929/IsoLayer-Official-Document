Step 2 — Verify the Recovery Mount

Before proceeding with initialization, we need to verify that the recovery partition was mounted correctly. IsoLayer's state engine indexes all directories in the recovery mount to build its snapshot registry.

Why this matters: The inspection file is used by IsoLayer's initialization routine in the next step. If the mount is misconfigured or empty, the initializer will fail with a RECOVERY_MOUNT_EMPTY error.
Generate the mount inspection manifest

List all directories inside the recovery mount and save them to an inspection file. IsoLayer uses this manifest to verify mount integrity and to register the existing state tree.

# Generate the recovery mount inspection manifest
ls -d /mnt/recovery/*/ > /mnt/recovery/inspect.txt

# Verify the manifest was created
cat /mnt/recovery/inspect.txt

You should see a list of directories. If inspect.txt is empty, go back to Step 1 and verify your Docker mount command.

Once you've confirmed the manifest contains directory entries, you're ready for the final initialization step.
