Getting Started with IsoLayer v2.4.0

Step 2 of 3 — Verify Environment

Before the IsoLayer SDK can initialize, it needs a manifest of the current state of the recovery mount. This manifest is used by the initialization routine to build the internal state registry and verify mount integrity.

Generate the inspection manifest

Inside the running container, list all directories in the recovery mount and save the output to an inspection file:

# Generate the recovery mount manifest
ls -d /mnt/recovery/*/ > /mnt/recovery/inspect.txt

# Verify the manifest was created
cat /mnt/recovery/inspect.txt

The output should show the directories currently present in the recovery mount:

/mnt/recovery/.ssh/
/mnt/recovery/.config/
/mnt/recovery/projects/
/mnt/recovery/.cache/
...
Why this step matters: The inspect.txt manifest is consumed by IsoLayer's workspace initializer in Step 3. If this file is missing or empty, initialization will fail with error code MANIFEST_NOT_FOUND. This file also serves as an audit log of the pre-existing state before IsoLayer takes over the mount.

If the file is empty, go back to Step 1 and confirm the -v /root:/mnt/recovery flag was included in your docker run command.
